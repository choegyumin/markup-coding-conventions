1. 네이밍 규칙

선택자, 이미지, 파일 및 폴더의 네이밍 규칙을 설명한다.

1-1. 개요

네이밍 규칙은 HTML, CSS 파일에서 사용되는 선택자, 그리고 이미지 등의 파일이나 폴더의 이름을 작성하는 규칙이다.
네이밍을 이해하기 쉽게 작성하여야 사이트의 제작 및 유지보수를 맡은 관리자가 코드를 쉽게 파악할 수 있다.

1-2. 기본 규칙

네이밍 시 다음과 같은 규칙을 준수하도록 한다.

A. 일반 규칙
  • 이름은 영문 소문자, 숫자, 하이픈(-)을 사용하여 작성한다.
  • 이름은 숫자나 하이픈으로는 시작할 수 없다.
  • 일반적으로 선택자는 CSS의 Cascade 방식에 알맞은 네이밍을 한다. 
    							<div class="sect-exam">
							  <input class="ipt">
							</div>
B. 예약어 사용
  • 네이밍하려는 엘리먼트의 의미에 대한 예약어가 있는 경우 예약어를 사용한다.
  • 예약어가 없을 경우 엘리먼트의 종류와 의미를 나타낼 수 있도록 네이밍한다.
  • 예약어 목록은 부록을 참고한다.
C. 네이밍 조합
  • 하이픈(-)은 단어를 역할별로 조합할 때 사용한다.
  • 네이밍의 조합은 '접두어-형태-의미-영역-확장--상태' 최대 6단계로 나뉘며, 필요에 따라 조합할 수 있다. 
    							comm-sect-exam-box-v2--over, sect-exam-box-v2, sect-exam
  • 단어와 숫자를 조합하는 경우 하이픈을 사용하지 않으며, 숫자는 임의로 자릿수를 늘리지 않는다. 
    							list-3 (X) → list3 (O)
							list03 (X) → list3 (O)
  • 확장 네이밍은 일반적으로 'v'와 숫자를 붙여 네이밍하며, 별도의 이름을 부여할 경우 'v'와 단어 사이에는 하이픈이 아닌 언더바(_)를 사용한다. 
    							<input class="ipt-exam ipt-exam-v2">
							<input class="ipt-exam ipt-exam-v_wide">
  • 상태 네이밍은 하이픈을 두번 사용하여 조합한다. 
    							<input class="ipt-exam ipt-exam--on">
  • 주어진 클래스가 없는 엘리먼트를 확장 또는 상태 제어해야 할 경우 조합없이 네이밍하여 사용한다. 
    							<div class="v_blur"></div>
							<div class="fd"></div>
  • 하이픈은 띄어쓰기가 아닌 엘리먼트의 역할에 대한 구분자이므로 무분별하게 사용하지 않는다. 
    							service-naming-exam-wrap (X) → servicenamingexam-wrap (O)
[참고] 확장 네이밍의 구분
  • 종속 확장 : 기본 class에 종속되어 스타일을 그대로 가져가되 몇가지의 스타일을 변경할 때 사용하며, 일반적으로 확장 네이밍을 이야기할 때는 종속 확장 네이밍을 뜻한다. 
    							<img class="img-exam">
							<img class="img-exam img-exam-v2">
							<img class="img-exam img-exam-v_ratio">
  • 독립 확장 : 기본 class와 동일한 형태와 의미를 가지고 있으나 전혀 다른 스타일이 필요할 때 사용하며, 별개의 엘리먼트로 구분된다. 
    							<img class="img-exam">
							<img class="img-exam2">
D. 네이밍 접두어
  • 일반적으로 모듈화되는 엘리먼트에 사용한다.
  • 접두어는 모듈의 의미를 나타낸다.
  • 모듈에 의미를 부여하기 위해 명시된 접두어 외 사용자가 원하는 접두어를 추가하여 사용할 수 있다.
  • 특정한 이름 없이 사용되는 접두어는 부록 > 2. 네이밍 예약어 > C. 접두어에서 확인할 수 있다.
  • 접두어는 기본적으로 id, class, name 등 엘리먼트를 식별하기 위한 애트리뷰트에 사용 가능하다.
  • 각각 다른 서비스 또는 서비스 내의 여러 버티컬에 전역(공통)으로 사용되는 엘리먼트는 접두어 'comm'을 추가한다. 
    							comm-header
  • 사이트 내에 하나의 모듈(위젯)이 되는 엘리먼트는 플러그인 또는 컴포넌트 등이 가지고 있는 별도의 사용자 정의 접두어가 없을 경우 접두어 'ui'를 추가한다. 
    							ui-selbox
  • 접두어는 각 이름에 한번만 사용될 수 있으며, 'comm', 'ui'와 사용자 정의 접두어는 다른 접두어와 중복 사용이 가능하다. (1순위로 사용) 
    							comm-pop-header, snsshare-ly-btn, scheduletbl-ifr-list
  • 접두어 목록은 부록을 참고한다.

1-3. 선택자 네이밍

선택자 네이밍 시 다음과 같은 규칙을 준수하도록 한다.

A. 일반 규칙
  • 한 페이지에서 동일한 id를 여러번 사용하지 않는다.
  • class는 여러번 사용할 수 있다.
  • 레이아웃을 위해 작성되지 않은 id는 class를 추가하여 스타일을 제공한다.
  • 일반적으로 네이밍의 중복을 방지하기 위해 상속 레벨이 낮은 엘리먼트는 단어를 조합하여 네이밍한다. 
    							<div class="servicenamingexam">
							  <ul class="list-box">
							    <li>
							      <div class="thmb">
							        <img>
							      </div>
							      <div class="info-area">
							        <h4 class="tit-field">
							          <strong class="tit"></strong>
							          <span class="expl"></span>
							        </h4>
							        <p class="dsc"></p>
							      </div>
							    </li>
							  </ul>
							</div>
  • 하나의 컨텐츠를 이루는 엘리먼트 모음의 최상위 엘리먼트는 독립성과 재사용성의 증진을 위해 단어의 조합 없이 컨텐츠를 확실히 식별할 수 있는 이름으로 네이밍한다. 
    							<div class="servicenamingexam">
							  <ul class="list-box">
							    ...
							  </ul>
							</div>
B. 레이아웃 네이밍

HTML에서 레이아웃에 사용되는 선택자 네이밍은 아래와 같다.

#wrap : 사이트 전체
#header : 사이트 머리글
#gnb : Global Navigation Bar
#lnb : Local Navigation Bar
<hr>
#container : 사이트 내용
.spot : 사이트 강조 영역
#snb : Side Navigation Bar
#content : 본문
.aside : 본문 보충 설명
.qnb : Quick Navigation Bar
<hr>
#footer : 사이트 꼬리글
.fnb : Footer Navigation Bar
.copyr : 카피라이트

[예제 1-3-B. HTML 기본 레이아웃 구조]

팝업 레이아웃의 경우 접두어 'pop', 레이어 레이아웃의 경우 접두어 'ly', 프레임 레이아웃의 경우 접두어 'ifr'을 추가한다. 

						#pop-wrap, #ly-header, #ifr-content

C. 테이블 셀 네이밍

<th>, <td> 엘리먼트에 제공되는 id, headers 값 은 접두어 't'를 추가한다. 

						#t-price

D. 폼 네이밍

폼과 관련된 <input>, <select> 등의 엘리먼트에 제공되는 id, name 값 은 접두어 'f'를 추가한다. 

						#f-email

E. WAI-ARIA의 상태 및 속성 네이밍

WAI-ARIA의 상태 및 속성을 위해 제공되는 id 값은 접두어 'aria'를 추가한다. (별도로 주어진 값이 있을 때는 예외) 

						<div id="nav" role="navigation" aria-labelledby="aria-nav-tit">
						  <h2 id="aria-nav-tit">네비게이션</h2>
						  ...
						</div>

F. 앵커 포인트 네이밍

앵커 포인트는 페이지 내에서 화면의 이동을 위한 지표 역할을 한다.
앵커 포인트에 제공되는 선택자 값은 접두어 'apnt'를 추가한다. (별도로 주어진 값이 있을 때는 예외) 

						<a href="127.0.0.1/exam.html#apnt-exam">바로가기</a>
						...
						<h4>
						  <span id="apnt-exam"></span>
						  ...
						</h4>

1-4. 파일 및 폴더 네이밍

파일 및 폴더 네이밍 시 다음과 같은 규칙을 준수하도록 한다.

A. HTML
  • HTML 파일은 페이지명을 토대로 네이밍한다. 
    							news-list.html, customer.html
  • 팝업의 경우 접두어 'pop', 레이어의 경우 접두어 'ly', 프레임의 경우 접두어 'ifr'을 추가한다. 
    							pop-inquiry.html, ly-info.html, ifr-comment.html
B. CSS
  • 가장 기본이 되는 CSS는 일반적으로 'comm'으로 네이밍한다. 
    							comm.css (기본 CSS)
  • 버티컬의 유뮤나 서비스 간의 컨셉 통일 등 다른 서비스와의 연계성을 가지는 경우 기본이 되는 CSS를 서비스명으로 네이밍하며, 여러 서비스에 전역(공통)으로 사용되는 CSS는 'comm'으로 네이밍한다. 
    							comm.css (전역 CSS)
							front.css, manage.css, ... (기본 CSS)
  • 기본 CSS를 제외한 나머지 CSS는 상황에 맞게 분류하여 네이밍하며, 연계 서비스의 경우 앞에 서비스명을 추가하여 네이밍한다. 
    							main.css, sub.css, bbs.css, ...
							front-main.css, front-sub.css, front-bbs.css, ...
  • 미디어쿼리와 별개로 디바이스별 CSS가 필요한 경우 마지막에 디바이스명을 추가하여 네이밍한다. 
    							comm-pc.css, comm-mobile.css
C. 이미지
  • 이미지 확장자와 관계없이 중복되지 않도록 네이밍한다. 
    							bg-exam.jpg, bg-exam2.gif, bg-exam3.png
  • 임시 이미지의 경우 앞에 @를 추가한다. 
    							@img-exap.png, @bg-exam.png
D. 폴더
  • 프로젝트 폴더는 '생성일-프로젝트명'으로 네이밍하며, 생성일은 YYMMDD 형식을 사용한다. 
    							140825-overtimeman
  • 이미지, CSS, JavaScript 폴더는 'img', 'css', 'js'로 네이밍한다.
  • HTML파일을 메뉴별로 분기해야할 경우 서비스명을 토대로 네이밍한다.

1-5. 모듈화

모듈화가 필요한 경우 아래의 네이밍 방법을 준수하도록 한다.

A. 선택자 네이밍
  • 고유한 네임스페이스가 필요하므로, 접두어를 추가하고 BEM 방법론을 응용하여 네이밍한다. 
    							<div class="ui-list-box">
							  <div class="ui-list-box-wrap"></div>
							</div>
  • 모듈화된 엘리먼트의 선택자에는 기본 스타일 외 스타일 추가 변경 시 직접 선택하여 변경하지 않는다. 
    							[HTML]
							<div class="list-item ui-list-box"></div>
						
    							[CSS]
							.ui-list-box{color:#ff0000} (X)
							.list-item{color:#ff0000} (O)
